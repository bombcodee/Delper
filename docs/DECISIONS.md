# Delper - 결정 사항 로그

## 아키텍처 결정

### 프레임워크 없이 Vanilla JS 사용
- **결정**: React/Vue 대신 순수 HTML/CSS/JS
- **이유**: 정적 가이드 문서에 프레임워크는 오버엔지니어링. GitHub Pages에 바로 배포 가능. 빌드 단계 불필요.

### ES Modules 대신 글로벌 함수 사용
- **결정**: `import/export` 대신 `<script>` 태그로 로드, 전역 함수
- **이유**: `file://` 프로토콜에서 ES Modules CORS 문제. 심플한 구조에 모듈 시스템 불필요.

### 데이터를 JS 파일로 분리 (JSON 아님)
- **결정**: `glossary-data.js`에서 `const GLOSSARY_DATA = [...]` 형태
- **이유**: JSON이면 fetch()가 필요하고 file:// 에서 CORS 문제. JS 파일이면 script 태그로 로드 즉시 사용 가능.

## 디자인 결정

### 테마 5개 (Dark, Midnight, Spring, Blossom, Sky)
- **결정**: 다크 2종 + 라이트 3종
- **이유**: 사용자가 "블랙+그레이 모던 + 봄라이트 부드러운 색감 + 하늘색" 요청. 다크 1개로는 취향 차이 커버 불가.

### Delper 로고에 그라데이션 텍스트
- **결정**: `var(--purple) → var(--blue)` 그라데이션, 테마별 자동 변경
- **이유**: 단색보다 눈에 띄면서 촌스럽지 않은 방법. CSS 변수 사용으로 테마 연동.

### 체크리스트 취소선 UX
- **결정**: 체크박스만 아니라 문장 어디든 클릭 → 취소선
- **이유**: 사용자가 "체크박스 말고 문장 아무곳이나 클릭하면 중단줄이 쫙 그어지면서 완료표시" 요청.

### 북마크 리본 스타일
- **결정**: 카드 우측 상단 리본/깃발 모양 (CSS clip-path)
- **이유**: 단순 별표 아이콘보다 시각적으로 강조. 옵션 A(원형 배경)와 B(리본) 중 사용자가 B 선택.

### 퍼널 단계 카드화
- **결정**: 각 퍼널 단계에 border + box-shadow 추가
- **이유**: 배경색만으로는 단계 구분이 불명확. 테두리로 명확한 구분.

### KPI 테이블에 개선 방법 컬럼 추가
- **결정**: 기존 4열(지표/의미/건강/위험) → 5열(+개선 방법)
- **이유**: 숫자만 보여주면 "그래서 어떻게 개선하지?"에 답이 없음. 실용성 강화.

### 수익 모델 의사결정 트리 4칸
- **결정**: 3칸 → 4칸 (플랫폼/커뮤니티 추가)
- **이유**: 3칸에서 텍스트 줄바꿈 발생. 4칸으로 콘텐츠 분산 + 유형 커버리지 확대.

## 네이밍 결정

### 프로젝트 이름: Delper
- **결정**: Delper (Develop + Helper)
- **대안 검토**: DevForge, ShipKit, BuildPal
- **이유**: 짧고 발음 자연스럽고 의미 직관적. "Helper" 뉘앙스가 가이드 성격에 적합.

### localStorage 키: delper-theme, delper-bookmarks
- **결정**: `dev-guide-*` → `delper-*` 로 변경
- **이유**: 브랜딩 일관성.

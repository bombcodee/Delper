## 1. API (Application Programming Interface)

**프로그램끼리 대화하는 약속된 방법**

### 비유

```
식당에서:
  손님(당신) → 메뉴판 보고 주문 → 웨이터(API) → 주방(서버) → 음식 가져옴

프로그래밍에서:
  당신의 코드 → 정해진 형식으로 요청 → API → 상대방 서버 → 결과 돌려줌
```

### 실제 예시

**날씨 API:**
```
요청: "서울 날씨 알려줘"
→ https://api.weather.com/seoul?key=abc123
응답: { "temp": 18, "condition": "맑음" }
```

**Claude API:**
```
요청: "이 텍스트를 요약해줘"
→ https://api.anthropic.com/v1/messages
   { "model": "claude-haiku-4-5", "messages": [{"role": "user", "content": "요약해줘..."}] }
응답: { "content": "요약 결과..." }
```

### REST API 기본 동작

| 메서드 | 의미 | 비유 |
|--------|------|------|
| **GET** | 데이터 조회 | "이거 보여줘" |
| **POST** | 데이터 생성 | "이거 새로 만들어줘" |
| **PUT** | 데이터 수정 | "이거 바꿔줘" |
| **DELETE** | 데이터 삭제 | "이거 지워줘" |

## 2. API 비용 구조

### 왜 돈이 드는가?

```
당신이 "요약해줘" 요청
    ↓
Anthropic 데이터센터의 GPU가 수십억 개 파라미터를 계산
    ↓
답변 생성 (전기, 냉각, 서버 유지비 발생)
    ↓
비용을 요청한 사람에게 청구
```

### 토큰이란?

AI API의 과금 단위. 글자 수와 비슷하지만 정확히 같지는 않음.

```
대략적 기준:
- 영어: 1단어 ≈ 1~2 토큰
- 한글: 1글자 ≈ 1~2 토큰
- "안녕하세요" ≈ 3~5 토큰
```

### AI 모델 티어 비교

| | Haiku (싼 것) | Sonnet (중간) | Opus (비싼 것) |
|---|---|---|---|
| **가격 (100만 토큰당)** | 입력 $0.80 / 출력 $4 | 입력 $3 / 출력 $15 | 입력 $15 / 출력 $75 |
| **성능** | 간단한 분류, 요약 | 코딩, 분석 | 복잡한 추론, 창의적 작업 |
| **속도** | 가장 빠름 | 보통 | 가장 느림 |
| **비유** | 인턴 (빠르고 싸지만 단순 작업) | 주니어 개발자 | 시니어 아키텍트 |

### 용도별 모델 선택

| 용도 | 추천 모델 | 이유 |
|------|----------|------|
| 텍스트 분류 ("이건 용어야? 리소스야?") | Haiku | 단순 판단, 비용 최소화 |
| 코드 생성, 리팩토링 | Sonnet | 균형 잡힌 성능 |
| 아키텍처 설계, 복잡한 분석 | Opus | 최고 품질 필요 |
| Quick Add 분류 (Delper) | Haiku | 월 $1 미만으로 충분 |

## 3. 서버와 클라우드

### 서버 = 24시간 켜져있는 컴퓨터

```
당신의 노트북도 서버가 될 수 있음
→ 하지만 24시간 안 끄고 인터넷에 연결해둬야 함
→ 끄면 서비스 중단
→ 그래서 전문 서버를 씀
```

### 클라우드 = 남의 컴퓨터를 빌려 쓰는 것

```
서버 직접 운영 = 내 집에 컴퓨터 놓고 24시간 켜둠
클라우드       = PC방 컴퓨터를 빌려서 씀 (내가 관리 안 해도 됨)
```

### 비교

| | 자체 서버 | 클라우드 |
|---|---|---|
| **비용** | 장비 구매 + 전기세 + 인터넷 | 월 사용료 (종량제) |
| **관리** | 직접 (고장, 보안, 업데이트) | 업체가 대신 |
| **확장** | 새 장비 구매 필요 | 클릭 몇 번으로 |
| **안정성** | 정전되면 끝 | 여러 곳에 백업 |
| **비유** | 자가용 | 택시/렌트카 |

### 주요 클라우드 서비스

| 서비스 | 운영사 | 특징 |
|--------|--------|------|
| **AWS** | Amazon | 가장 큼, 모든 기능 있음, 복잡함 |
| **GCP** | Google | AI/ML 강점, BigQuery |
| **Azure** | Microsoft | 기업용, Windows 연동 |
| **Cloudflare** | Cloudflare | CDN + Workers, 심플, 무료 많음 |
| **Vercel** | Vercel | 프론트엔드 배포 특화, Next.js |

## 4. Cloudflare Workers

### 정의

내 코드를 전 세계 300+곳에 뿌려주는 서비스

### 일반 서버 vs Workers

```
일반 서버:
  서울에 컴퓨터 1대
  → 한국 사용자: 빠름
  → 미국 사용자: 느림 (태평양 왕복)

Workers:
  전 세계 300곳에 코드가 복사됨
  → 한국 → 서울 컴퓨터가 응답
  → 미국 → LA 컴퓨터가 응답
  → 유럽 → 런던 컴퓨터가 응답
```

**비유: 본점 1개(일반 서버) vs 전국 편의점 체인(Workers)**

### "초소형"의 의미

```
일반 서버: 컴퓨터 1대를 통째로 빌림 → 24시간 비용 발생
Workers:   함수 1개만 올림 → 요청 올 때만 실행 → 안 쓰면 비용 0원
```

**비유: 사무실 임대(24시간 월세) vs 공유 오피스 회의실(쓸 때만 결제)**

### 서버리스(Serverless) 개념

```
"서버가 없다"가 아니라 "관리할 서버가 없다"
→ 서버는 Cloudflare가 관리
→ 나는 코드(함수)만 올리면 됨
→ 서버 보안, 업데이트, 장애 대응 전부 Cloudflare 몫
```

### 무료인 이유

```
비즈니스 모델: 무료로 유입 → 트래픽 커지면 유료 전환 유도

무료 범위:
- 일 10만 요청
- 10ms CPU 시간
→ 개인 프로젝트에서는 영원히 무료 수준
```

### Delper에서의 역할

```
사이트 JavaScript에 API 키를 넣으면 → 누구나 F12로 볼 수 있음 (위험)
Worker에 API 키를 넣으면 → 외부에서 절대 못 봄 (안전)

Worker = "API 키를 안전하게 숨기면서 Claude를 호출하는 중간 다리"

비유:
  키 직접 노출 = 현관 비밀번호를 문에 적어놓음
  Worker 사용 = 경비실에 비밀번호를 맡기고, 경비원이 대신 문 열어줌
```

## 5. GitHub의 역할

### 단순 코드 업로드가 아닌 "버전 관리"

```
버전 관리 = 모든 수정 이력을 기록
→ 3일 전 코드로 되돌리기 가능
→ 누가 뭘 언제 바꿨는지 추적 가능
→ 여러 사람이 동시에 작업해도 충돌 방지
```

### GitHub Pages

```
GitHub 저장소의 파일 → 웹사이트로 자동 변환
bombcodee/Delper 저장소 → bombcodee.github.io/Delper 사이트

파일이 바뀌면 → 자동으로 사이트에 반영
```

### GitHub API

```
웹에서 마우스로 하는 걸, 코드로 하는 것

방법 1 (수동): GitHub.com → 파일 클릭 → 연필 아이콘 → 수정
방법 2 (API):  코드에서 "이 파일을 이렇게 바꿔줘" 요청
```

### PR (Pull Request)

```
"이렇게 수정하려는데 괜찮아?" 라고 물어보는 것

직접 수정: 바로 고침 → 실수하면 바로 반영됨 (위험)
PR로 수정: 제안 → 확인 → 승인 후 반영 (안전)

비유:
  직접 수정 = 벽에 바로 페인트칠
  PR = "이 색으로 칠할 건데 어때?" 샘플 보여주기
```

### GitHub Actions

```
특정 조건에서 자동으로 코드를 실행하는 서비스

예시:
- "매주 월요일에 이 스크립트 실행해라"
- "누가 PR 만들면 자동으로 테스트 돌려라"
- "main 브랜치에 푸시하면 자동으로 배포해라"

비유: 무인 편의점 알바 (조건 맞으면 자동으로 일함)
```

### GitHub Secrets

```
API 키를 안전하게 저장하는 금고

GitHub Actions에서 코드를 실행할 때
→ Secrets에 저장된 키를 사용
→ 코드에는 키가 안 보임
→ 저장소가 공개여도 키는 비공개
```

## 6. 데이터 전송의 물리적 원리

### 유선 통신: 광케이블

```
광케이블 = 유리 섬유 안에서 빛이 왔다 갔다
→ 빛의 속도로 데이터 전송
→ 서울 → LA: 약 0.05초 (50ms)
```

**해저 광케이블:**
```
대륙 간 통신의 실체는 바다 밑에 깔린 케이블

한국 ──해저──→ 일본 ──태평양 해저──→ 미국

진짜 물리적인 줄. 수만 km.
가끔 상어가 물어뜯거나 닻에 걸려 끊기기도 함.
```

### 무선 통신: 전파

```
전파 = 전자기파 = 빛과 같은 종류 (다만 눈에 안 보임)

소리: 공기를 진동시켜서 정보 전달 → 사람 귀가 받음
전파: 전자기장을 진동시켜서 정보 전달 → 안테나가 받음
```

**Wi-Fi 원리:**
```
라디오: 방송국이 전파 쏨 → 라디오가 잡아서 소리로 변환
Wi-Fi:  공유기가 전파 쏨 → 노트북이 잡아서 데이터로 변환

완전히 같은 원리. 주파수와 데이터 형식만 다름.
```

## 7. 지금 이 순간 당신을 관통하는 전파들

```
이 방에 있는 전파들:
- Wi-Fi (2.4GHz, 5GHz)
- 블루투스 (이어폰, 키보드)
- 4G/5G (폰 기지국)
- FM 라디오
- TV 방송
- GPS 위성 신호
- 전자레인지 (2.4GHz — Wi-Fi랑 같은 주파수!)

→ 동시에 수십~수백 개의 전파가 겹쳐있음
```

### 인체에 영향?

```
위험한 전자기파: 엑스레이, 감마선 → 세포 DNA 파괴 가능 (이온화 방사선)
안전한 전자기파: 라디오, Wi-Fi, 5G → 에너지가 너무 약함 (비이온화 방사선)

비유:
  엑스레이 = 총알 (관통함)
  Wi-Fi    = 바람 (스쳐지나감)
```

현재 과학적 합의: **일상 수준에서 무해**

## 8. 전파 충돌과 해결

### 주파수로 구분

```
비유: 라디오 채널
  KBS = 93.1 MHz
  MBC = 95.9 MHz
  SBS = 107.7 MHz
  → 다 동시에 방송하지만, 채널이 다르니 안 섞임
```

| 기술 | 주파수 대역 |
|------|------------|
| Wi-Fi | 2.4GHz, 5GHz |
| 블루투스 | 2.4GHz |
| 5G | 3.5GHz, 28GHz |
| FM 라디오 | 88~108MHz |
| GPS | 1.575GHz |

### 같은 주파수일 때 해결 방법

**시분할 (TDMA):**
```
"0.001초는 내 차례, 다음 0.001초는 네 차례"
→ 너무 빨라서 사람은 동시에 느낌
```

**코드 분할 (CDMA):**
```
모든 신호에 고유 암호를 섞어서 보냄
수신자는 자기 암호에 맞는 것만 골라냄

비유:
  100명이 동시에 말하는 방에서
  한국어, 영어, 일본어가 섞여있어도
  한국어 아는 사람은 한국어만 골라 들을 수 있음
  → 각 "언어"가 고유 코드 역할
```

### 실제 충돌이 일어나는 경우

```
아파트에서 옆집/윗집/아랫집 전부 Wi-Fi 쓰면
→ 같은 채널에 몰리면 서로 "순서 대기" → 느려짐
→ 데이터가 깨지면 자동으로 "다시 보내줘" 재전송
→ 데이터가 사라지진 않고, 느려지는 것
```

## 9. 자연환경의 영향

| 환경 | 영향 | 설명 |
|------|------|------|
| **바람** | ❌ 없음 | 전파는 물체가 아님. 빛이 바람에 안 날아가는 것과 같음 |
| **벽** | ⚠️ 약해짐 | 콘크리트 > 나무 > 유리 순으로 차단 |
| **비/눈** | ⚠️ 일부만 | 28GHz 이상 초고주파만 약간 영향 |
| **금속** | 🚫 반사 | 전자레인지가 금속 벽으로 전파 가두는 원리 |
| **태양 폭풍** | ⚠️ 드묾 | GPS, 위성통신에 영향 (수년에 한 번) |

## 10. 위성 인터넷 (스타링크)

### 기본 원리

```
스타링크 위성 고도: 약 550km
서울 → 부산: 325km
→ 그 1.7배 높이에 위성이 떠있음

전파가 빛의 속도(초속 30만km)로 날아감
→ 550km = 0.002초면 도착
```

**비유: 손전등을 하늘로 비추면 빛이 올라가는 것과 같음. 전파도 같은 전자기파니까 위로 쭉 날아감. 다만 눈에 안 보일 뿐.**

### 전체 경로

```
산 속 노트북
    ↓ 전파 (550km 위로)
스타링크 위성
    ↓ 위성끼리 레이저로 전달 (우주에서)
지상 기지국 근처 위성
    ↓ 전파 (550km 아래로)
지상 기지국
    ↓ 광케이블 (유선)
인터넷
```

위성 = "하늘에 있는 공유기" 역할

## 11. 실제 데이터 전송 경로 예시

Delper Quick Add 버튼을 누르면 일어나는 일:

```
1. [노트북]
   → Wi-Fi 전파로 공유기에 쏨 (무선, 수 미터)

2. [공유기]
   → 유선 케이블로 통신사에 전달 (유선)

3. [통신사 (KT/SKT 등)]
   → 광케이블로 서울 데이터센터에 전달 (유선)

4. [Cloudflare 서울 데이터센터]
   → Worker가 요청 처리
   → Claude API 호출 (광케이블로 미국까지)

5. [Anthropic 서버 (미국)]
   → Claude가 분류 처리
   → 결과를 다시 광케이블로 반환

6. 역순으로 노트북까지 도착

전체 과정: 약 0.5~2초
```
